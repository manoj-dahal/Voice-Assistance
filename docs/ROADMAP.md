# ERA Roadmap

This roadmap is directional. Only features labeled Available in the running application should be treated as implemented.

## Phase 1 — Local intelligence

- local embedding index for notes and projects;
- editable knowledge-graph relationships;
- memory provenance and revision history;
- encrypted workspace export and restore;
- durable scheduled reminders beyond browser-local timers.

## Phase 2 — Desktop agent

- operating-system application discovery adapters;
- permission-scoped launching;
- global command shortcut;
- system tray and quick-capture overlay;
- safe terminal profiles;
- clipboard intelligence disabled by default.

## Phase 3 — Voice

- local wake phrase;
- voice activity detection;
- streaming full-duplex provider audio;
- interruption handling;
- multilingual code-switching;
- pronunciation dictionary.

## Phase 4 — Connectors

Read-only first:

1. Google Calendar;
2. Gmail summaries and draft-only replies;
3. GitHub repositories and checks;
4. selected Notion pages;
5. weather and air quality.

Every connector needs revocation, audit, data minimization, and permission tests.

## Phase 5 — Skills and MCP

- signed plugin packages;
- isolated workers;
- MCP discovery and connection approval;
- skill versioning and permission diffs;
- execution receipts;
- conflict detection;
- marketplace revocation.

## Phase 6 — Device handoff

- encrypted desktop/phone pairing;
- remote workflow approval;
- offline mobile capture;
- explicit phone-camera sessions;
- per-device capability matrix.

## Research-only areas

- speaker separation;
- health and biometric adapters;
- robotics;
- industrial systems;
- neural interfaces;
- volumetric and spatial displays.

## Speculative concepts

ERA does not claim consciousness, quantum acceleration, future knowledge, multiverse modeling, or independent sovereign authority. These remain concepts unless measurable implementations and evidence exist.
